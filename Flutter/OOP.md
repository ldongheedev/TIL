# 객체지향 프로그래밍

## 객체지향 프로그래밍이란?
Object Oriented Programming 이라 부르며 **프로그램을 '객체'라는 독릡된 단위로 나누어, 이 객체들이 서로 상호작용하며 동작하도록 설계하는 프로그래밍 패러다임**

## 핵심 개념
- 객체 Object   
현실 세계의 사물이나 개념을 코드로 표현한 것.   
- 클래스 Class   
객체를 만들기 위한 설계도.   
객체의 속성(변수)과 행동(메서드)을 정의.   
- 인스턴스 Instance   
클래스를 기반으로 실제로 생성된 객체.   
- 속성 Attribute   
객체가 가진 데이터.   
- 메서드 Methodd   
객체가 수행할 수 있는 행동


## OOP 4대 원칙
- **캡슐화** Encapsulation 
데이터와 기능을 하나로 묶고, 외부에서 접근하지 못하게 보호     
- **상속** Inheritance   
기존 클래스의 속상과 기능을 물려받아 새로운 클래스로 정의   
- **다형성** Polymorphism   
같은 이름의 메서드가 상황에 따라 다르게 동작 (ex: Animal.makeSound())
- **추상화** Abstraction   
복잡한 구현을 숨기고, 필요한 기능만 드러냄.

mixin : 아이템과 같이 생각하면 됨. (이미 구현된 기능을 줌)
인터페이스(implements) : 상속 관계가 아닌 객체들을 묶어줌. (묶이면 기능을 반드시 구현해야 함) // 매 번 구현해 주어야 함
